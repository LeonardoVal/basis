basis

Basic utility functions and definitions used in Creatartis' projects.

callStack(error=none):

Returns an array with the callstack of error or (if missing) a new one is used, hence returning the current callStack.

new Chronometer(time):

Utility object for measuring time lapses.

Chronometer.chronometer(f, times=1):

Executes the parameterless function f the given number of times and logs the time each run takes. Returns the average time.

Chronometer.reset(time=now):

Resets the chronometer's to the given time or now by default.

Chronometer.tick():

Get the elapsed time since the creation or resetting of the chronometer, and resets it.

Chronometer.time():

Get the elapsed time since the creation or resetting of the chronometer.

copy(objTo, objFrom...):

Copies all own properties of the given objects into objTo, and returns it. If only one object is given, a copy of the objTo object is returned.

new Events(config):

Event handler that manages callbacks registered as listeners.

Events.emit(eventName, ...args):

Emits an event with the given arguments. Listeners' callbacks are called asynchronously.

Events.isOpen=true:

An open Events accepts listeners to any event. Otherwise event names have to be specified previously via the 'events' property in the configuration.

Events.listeners(eventName):

Returns an array with the listeners for the given event.

Events.maxListeners=Infinity:

Maximum amount of listeners these events can have.

Events.off(eventName, callback):

Deregisters the callback from the event.

Events.on(eventName, callback, times=Infinity):

Registers a callback to listen to the event the given number of times, or always by default.

Events.once(eventName, callback):

Registers a callback to listen to the event only once.

new Future():

An implementation of futures (aka deferreds or promises), a construction oriented to simplify the interaction between parallel threads. A future represents a value that is being calculated asynchronously. Callbacks are registered for when the value becomes available or an error raised.

Future.__complete__(context, value, state):

Internal use method that changes this future's state from pending to state, calling all the corresponding callbacks with the given context and value.

Future.__onCancel__(callback...):

Registers one or more callbacks to be called when this Future is cancelled.

Future.__register__(callback, state):

Registers a callbacks to be called when this Future is in the given state. If this Future is already in that state the callback is called right away. If this Future is neither pending nor in that state, the callback is ignored.

static Future.all(futures):

Returns a Future that is resolved when all the given futures are resolved, or rejected when one is rejected. If no futures are given, the result is resolved with [].

Future.always(callback...):

Registers one or more callbacks to be called when this Future is either resolved or rejected. It is the same as using done() and fail() functions.

static Future.any(futures):

Returns a Future that is resolved when any of the given futures are resolved, or rejected when all are rejected. If no futures are given, the result is rejected with undefined.

Future.bind(future):

Binds this future resolution, rejection and cancellation to the given future's corresponding resolution, rejection and cancellation.

Future.cancel(reason):

Marks the future as 'cancelled' and disregards all callbacks. This method may be called by either the producer or the consumer threads.

static Future.delay(ms, value):

Return a future that will be resolved with the given value after the given time in milliseconds. Time is forced to be at least 10ms. If value is undefined, the timestamp when the function is called is used.

Future.done(callback...):

Registers one or more callbacks to be called when this Future is resolved. If this Future is already resolved the callbacks are called right away. If this Future is neither pending nor resolved, callbacks are ignored.

static Future.doWhile(action, condition):

Perform the action until the condition fails. The action is first called without arguments, and afterwards is called with the previous value. The conditions is always called with the last value returned by action.

Both action and condition may return futures. The condition by default is the boolean conversion of the action's returned value.

Future.fail(callback...):

Registers one or more callbacks to be called when this Future is rejected. If this Future is already rejected the callbacks are called right away. If this Future is neither pending nor rejected, callbacks are ignored.

static Future.imports(...modules):

Builds a future that loads the given modules using RequireJS' require function, and resolves to an array of the loaded modules.

static Future.invoke(fn, _this, args...):

Calls the function synchronously, returning a future resolved with the call's result. If an exceptions is raised, the future is rejected with it.

Future.isCancelled:

Checks if this future's state is 'cancelled'.

Future.isPending():

Checks if this future's state is 'pending'.

Future.isRejected:

Checks if this future's state is 'rejected'.

Future.isResolved:

Checks if this future's state is 'resolved'.

Future.reject(reason, context=this):

Marks the future as 'rejected' and calls onRejected callbacks. This method should be called by the producer thread when its process is aborted with an error.

If there aren't any onRejected callbacks registered, an Error is raised.

This can be reason (if it is already an Error) or a new Error with reason as message.

Future.resolve(value, context=this):

Marks the future as resolved. This method should be called by the producer thread when its process is finished successfully.

static Future.retrying(f, t=10, delay=100ms, delayFactor=2, maxDelay=5min):

Calls the function f upto t times until it returns a value or a future that is resolved. Each time is separated by a delay that gets increased by delayFactor upto maxDelay.

static Future.sequence(xs, f=None):

Evaluates all values and futures in the iterable xs in sequence. If it is given, the function f is called for each value.

Future.state=0:

Current state of the Future. Pending is 0, resolved is 1, rejected is 2, cancelled is 3.

Future.STATES=['pending', 'resolved', 'rejected', 'cancelled']:

An array with labels for the Future's possible states.

Future.then(onResolved, onRejected):

Returns a new Future which is resolved when this future is resolved, and rejected in the same way. The given callbacks are used to calculate a new value to either resolution or rejection of the new Future object.

static Future.when(value):

Unifies asynchronous and synchronous behaviours. If value is a Future it is returned as it is. Else a resolved Future is returned with the given value.

static Future.whileDo(condition, action):

Similar to futures.doWhile, but evaluates the condition first.

Global:

Global scope of the current execution environment. Usually (window) in browsers and (global) under NodeJS.

new HttpRequest():

A wrapper for XMLHttpRequest.

HttpRequest.get(url, content, headers, user, password):

Shortcut for a request with the GET method.

HttpRequest.getJSON(url, content, headers, user, password):

Makes a GET request and parses the response text as JSON.

HttpRequest.getText(url, content, headers, user, password):

Makes a GET request and returns the response's text.

HttpRequest.post(url, content, headers, user, password):

Shortcut for a request with the POST method.

HttpRequest.postJSON(url, content, headers, user, password):

Makes a POST request with the content encoded with JSON.stringify().

HttpRequest.request(method, url, content, headers, user, password):

Opens the request with the given method at the given url, sends the contents and returns a future that gets resolved when the request is responded.

initialize(subject, args):

Returns a new Initializer for the subject.

new Initializer(subject={}, args={}):

Initializers are object builders, allowing the declaration of default values, type checks and coercions, and other checks.

Initializer.array(id, options):

Assigns the id property with an array. Options may include:

  • options.elementTypes: Required type of the array's elements.
  • options.length: Required length of the array.

Initializer.attr(id, options={}):

Assigns the id property, performing all necessary verifications. If options.overwrite is false, an error is raised if the subject already has the attribute defined. If options.ignore is true, no error is raised and the assignment is skipped instead.

Initializer.bool(id, options):

Assigns the id property with a truth value.

Initializer.func(id, options):

Assigns the id property with a function.

Initializer.get(id, options):

Gets the value for the given id. If it is missing, options.defaultValue is used as the default value if defined. Else an error is raised.

If options.type is defined, the value is checked to be a member of said type. If options.coerce is true, the value may be coerced.

The function option.check can check the value further. If defined it is called with the value, and is expected to raise errors on failed conditions.

Other options include:

  • options.regexp: the value is matched agains a regular expression.
  • options.minimum: the value has to be greater than or equal to this value.
  • options.maximum: the value has to be less than or equal to this value.

Initializer.integer(id, options):

Assigns the id property with an integer value.

Initializer.number(id, options):

Assigns the id property with a numerical value.

Initializer.object(id, options):

Assigns the id property with an object.

Initializer.string(id, options):

Assigns the id property with a string value.

iterable(x):

Returns an iterable, either if x is already one or builds one from it.

new Iterable(it):

Constructor of iterable objects. Those have a __iter__() method that returns an iterator function. This iterator returns the current value and advances each time it is called. When sequence ends, the iterator raises the iterables.STOP_ITERATION singleton exception.

This constructor supports strings (iterating over each caracter), arrays, objects (key-value pairs) and functions (assuming it is the iterator constructor). A value of null or undefined is not allowed. Everything else is assumed to be the only value of a singleton sequence.

Iterable.__iteratorFromArray__(array):

Returns a constructor of iterators of the given array.

Iterable.__iteratorFromObject__(obj):

Returns an iterator constructor going through obj's properties as returned by Object.keys(). Each element in the sequence is an array with the property key and the property value.

Iterable.__iteratorFromString__(str):

Returns a constructor of iterators of the str string.

Iterable.__iteratorSingleton__(x):

Returns an iterator constructor with a singleton sequence with the given value.

Iterable.all(predicate, strict=false):

Returns true if for all elements in the sequence the predicate returns true, or if the sequence is empty.

Iterable.any(predicate, strict=false):

Returns false if for all elements in the sequence the predicate returns false, or if the sequence is empty.

Iterable.catchStop(exception):

If the exception is STOP_ITERATION it does nothing. If it is not, the exception is thrown.

Iterable.chain(iterables...):

Returns an iterable that iterates over the concatenation of this and all the given iterables.

Iterable.count():

Counts the number of elements in the sequence.

Iterable.cycle(n=Infinity):

Returns an iterable that loops n times over the elements of this Iterable (or forever by default).

Iterable.EMPTY:

An empty Iterable.

Iterable.filter(filterFunction, mapFunction):

Returns an iterable of this iterable elements for which filterFunction returns true. If mapFunction is given it is applyed before yielding the elements.

Iterable.flatten():

Chains all the iterables in the elements of this iterable.

Iterable.foldl(foldFunction, initial):

Folds the elements of this iterable with foldFunction as a left associative operator. The initial value is used as a starting point, but if it is not defined, then the first element in the sequence is used.

Iterable.foldr(foldFunction, initial):

Folds the elements of this iterable with foldFunction as a right associative operator. The initial value is used as a starting point, but if it is not defined, then the first element in the sequence is used.

Warning! This is the same as doing a foldl in a reversed iterable.

Iterable.forEach(doFunction, ifFunction):

Applies the doFunction to all elements complying with ifFunction, and returns the last result.

This is similar to basis.Iterable.map, but it does not return another iterable.

Iterable.greater(evaluation=<cast to number>):

Returns an array with the elements of the iterable with greater evaluation.

Iterable.head(defaultValue):

Returns the first element. If the sequence is empty it returns the defaultValue, or raise an exception if one is not given.

Iterable.isEmpty():

Returns if the sequence has no elements.

Iterable.iterate(f, x, n=Infinity):

Returns an iterable that repeatedly applies the function f to the value x, n times (or indefinitely by default).

Iterable.join(sep):

Concatenates all strings in the sequence using sep as separator. If sep is not given, '' is assumed.

Iterable.last(defaultValue):

Returns the last element. If the sequence is empty it returns the defaultValue, or raise an exception if one is not given.

Iterable.lesser(evaluation=<cast to number>):

Returns an array with the elements of the iterable with lesser evaluation.

Iterable.map(mapFunction, filterFunction):

Returns an iterable iterating on the results of applying mapFunction to each of this iterable elements. If filterFunction is given, only elements for which filterFunction returns true are considered.

Iterable.max(n=-Infinity):

Returns the maximum element of all elements in the sequence, or -Infinity if the sequence is empty.

Iterable.min(n=Infinity):

Returns the minimum element of all elements in the sequence, or Infinity if the sequence is empty.

Iterable.pluck(member):

Shortcut for a map that extracts a member from the objects in the sequence.

Inspired by <http://underscorejs.org/#pluck>.

Iterable.product(iterables...):

Returns an iterable that iterates over the Cartesian product of this and all the given iterables; yielding an array of the values of each.

Iterable.range(from=0, to, step=1):

Builds an Iterable object with number from argument from upto argument to, with the given step. For example, basis.range(2,12,3) represents the sequence {2, 5, 8, 11}.

Iterable.repeat(x, n=Infinity):

Returns an iterable that repeats the element x n times (or forever by default).

Iterable.reverse():

Returns an iterable with this iterable elements in reverse order.

Warning! It stores all this iterable's elements in an array.

Iterable.sample(n, random=Randomness.DEFAULT):

Returns an iterable with n elements of this iterable randomly selected. The order of the elements is maintained.

Iterable.scanl(foldFunction, initial):

Folds the elements of this iterable with foldFunction as a left associative operator. Instead of returning the last result, it iterates over the intermediate values in the folding sequence.

Iterable.scanr(foldFunction, initial):

Folds the elements of this iterable with foldFunction as a right associative operator. Instead of returning the last result, it iterates over the intermediate values in the folding sequence.

Warning! This is the same as doing a scanl in a reversed iterable.

Iterable.sorted(sortFunction):

Returns an iterable that goes through this iterable's elements in order.

Warning! This iterable's elements are stored in an array for sorting.

Iterable.stop():

Raises the STOP_ITERATION exception. If used inside an iterator it breaks the iteration.

Iterable.STOP_ITERATION:

Error raised when an iterator has finished. It is catched by all Iterable's functions.

Iterable.sum(n=0):

Returns the sum of all elements in the sequence, or 0 if the sequence is empty.

Iterable.toArray(array):

Appends to the array the elements of the sequence and returns it. If no array is given, a new one is used.

Iterable.toObject(obj={}):

Takes an iterable of 2 element arrays and assigns to the given object (or a new one by default) each key-value pairs as a property.

Iterable.zip(iterables...):

Returns an iterable that iterates over this and all the given iterables at the same time; yielding an array of the values of each and stopping at the first one finishing.

new Logger(name, parent=Logger.ROOT, level="INFO"):

Constructor of logger objects, which handle logging capabilities in a similar (but greatly simplified) fashion that Log4J.

Logger.appendAsWorkerMessages(messageTag='log'):

Appender that posts the log entries with the web workers postMessage() function.

Logger.appendToConsole():

Appender that writes messages to console (using console.log).

Logger.appendToFile(filePath, flags='a', encoding='utf-8'):

Appender that writes the log entries to a file using NodeJS's file system module.

Logger.appendToHtml(htmlElement=document.body, maxEntries=all):

Appender that writes the log entries as paragraphs inside the given htmlElement. The number of entries can be limited with maxEntries.

Warning! Formatted entry text is assumed to be valid HTML and hence is not escaped.

Logger.debug(message...):

Make a new log entry with the given message and the DEBUG level.

Logger.defaultFormat(name, time, level, message):

Formats are used by appenders. This default format concatenates the log entry data in a string.

Logger.error(message...):

Make a new log entry with the given message and the ERROR level.

Logger.fatal(message...):

Make a new log entry with the given message and the FATAL level.

Logger.htmlFormat(tag='pre', cssClassPrefix='log_'):

Returns a format function similar to the default format, but in an HTML element with CSS styling support.

Logger.info(message...):

Make a new log entry with the given message and the INFO level.

Logger.LEVELS:

Logging levels to use with Loggers: TRACE, DEBUG, INFO, WARN, ERROR and FATAL. The default one is INFO. Each one has a shortcut method (name in lower case) to log directly in that level.

Logger.log(level, message...):

If the given level is greater than the current logger's level, a new entry is appended. The message results of a timestamp and the arguments.

static Logger.ROOT:

The root logger must be the final ancestor of all loggers. It is the default parent of the Logger constructor.

Logger.stats():

Gets the logger's Statistics objects, creating it if necessary.

Logger.trace(message...):

Make a new log entry with the given message and the TRACE level.

Logger.warn(message...):

Make a new log entry with the given message and the WARN level.

obj(key, value...):

Object constructor based on key-value pairs. Useful to build objects when keys come from expressions instead of identifiers or numerals, which makes it impossible to use the object literal notation.

objects:

Bundle of OOP related functions and definitions.

objects.addMember(constructor, key, value, force=false):

Adds the value as a member of the constructor's type. If the constructor already has a member with the given key, it is overriden only if force is true.

objects.addMembers(constructor, members, force=false):

Adds all own properties of members to the constructor's type, using addMember.

objects.declare(supers..., members={}):

Object oriented implementations, influenced by Dojo's. The first super is considered the parent. The following supers add to the returned constructor's prototype, but do not override. The given members always override.

See Dojo's declare.

objects.subconstructor(parent, constructor=<new constructor>):

Returns a new constructor (or the given constructor) with an adjusted prototype inheriting from the parent.

objects.unimplemented(cls, id):

Returns a function that raises an "unimplemented method" exception.

ObjectType.coerce(value):

Converts the value to this type, if possible and necessary. If it is not possible, it raises a TypeError. This is the default behaviour.

ObjectType.isCompatible(value):

Returns if the value is assignment compatible with this type. By default the only compatible values are the ones in the type itself. But it can be overriden to allow subtypes and coercions.

ObjectType.isType(value):

Checks if the value is an object, and an instance of the specified constructor of this type (if applies).

raise(message...):

Builds a new instance of Error with the concatenation of the arguments as its message and throws it.

raiseIf(condition, message...):

If the condition is true a new Error is built and risen like in basis.raise().

new Randomness(generator):

Pseudorandom number generator constructor, based on a generator function. This is a function that is called without any parameters and returns a random number between 0 (inclusive) and 1 (exclusive). If none is given the standard Math.random() is used.

Randomness.averagedDistribution(times):

Returns another Randomness instance based on this one, but generating numbers by averaging its random values a given number of times. The result is an aproximation to the normal distribution as times increases.

By default times = 2 is assumed.

Randomness.choice(xs):

Randomnly selects an element from the iterable xs. If more than one is given, the element is chosen from the argument list.

Randomness.choices(n ,xs):

Randomnly selects n elements from xs. If more than two arguments are given, the arguments from 1 and on are considered as xs.

static Randomness.DEFAULT:

Default static instance, provided for convenience. Uses Math.random().

static Randomness.linearCongruential(m, a, c):

Returns a pseudorandom number generator constructor implemented with the linear congruential algorithm.

See <http://en.wikipedia.org/wiki/Linear_congruential_generator>.

static Randomness.linearCongruential.numericalRecipies(seed):

Builds a linear congruential pseudorandom number generator as it is specified in Numerical Recipies. See <http://www.nr.com/>.

static Randomness.linearCongruential.numericalRecipies(seed):

Builds a linear congruential pseudorandom number generator as it used by Borland C/C++.

static Randomness.linearCongruential.numericalRecipies(seed):

Builds a linear congruential pseudorandom number generator as it used by glibc. See <http://www.mscs.dal.ca/~selinger/random/>.

Randomness.random(x, y):

Called without arguments returns a random number in [0,1). Called with only the first argument x, returns a random number in [0, x). Called with both arguments return a random number in [x,y).

Randomness.randomBool(p=0.5):

Returns true with a probability of p, else false. By default p = 0.5 is assumed.

Randomness.randomInt(x, y):

Same as with Randomness.random(x,y) but returns integers instead.

Randomness.randoms(n, x, y):

Builds an array of n random numbers calling Randomness.random(x, y).

Randomness.shuffle(xs):

Randomnly rearranges elements in xs. Returns a copy.

Randomness.split(n, xs):

Take n elements from xs randomnly. Returns an array [A,B] with A being the taken elements and B the remaining. If more than two arguments are given, elements are taken from the second argument on.

Randomness.weightedChoices(n, weightedValues):

Chooses n values from weighted values randomly, such that each value's probability of being selected is proportional to its weight. The weightedValues must be an iterable of pairs [weight, value].

Weights are normalized, but if there are negative weights, the minimum value has probability zero.

new Statistic(keys):

Statistical logger object, representing one numerical value.

Statistic.accumulation(keys):

Creates a new Statistic that accumulates all that apply to the given keys.

Statistic.add(value, data=none):

Updates the statistics with the given value. Optionally data about the instances can be attached.

Statistic.addAll(values, data=none):

Adds all the given values (using this.add()).

Statistic.addStatistic(stat):

Adds the values in the given Statistic object to this one.

Statistic.addTick(data=undefined):

Adds to this statistic the time since startTime was called, and resets the chronometer.

Statistic.addTime(data=undefined):

Adds to this statistic the time since startTime was called.

Statistic.applies(keys):

Checks if all the given keys are this statistic's keys.

Statistic.average():

Calculates the current average, or zero if values have not been added.

Statistic.count():

Get the current count, or 0 if values have not been added.

Statistic.DEFAULT_GAIN_FACTOR=0.99:

Default factor used in the gain() method.

Statistic.gain(value, factor=DEFAULT_GAIN_FACTOR, data=none):

Like add, but fades previous values by multiplying them by the given factor. This is useful to implement schemes similar to exponential moving averages.

Statistic.gainAll(values, factor=DEFAULT_GAIN_FACTOR, data=none):

Gains all the given values (using this.gain()).

Statistic.maxData():

Get the data associated with the current maximum, or undefined if there is not one.

Statistic.maximum():

Get the current maximum, or -Infinity if values have not been added.

Statistic.minData():

Get the data associated with the current minimum, or undefined if there is not one.

Statistic.minimum():

Get the current minimum, or Infinity if values have not been added.

Statistic.reset():

Resets the statistics, and returns this object for chaining.

Statistic.squareSum():

Get the current sum of squares, or zero if values have not been added.

Statistic.standardDeviation(center=average):

Calculates current standard deviation, as the square root of the current variance.

Statistic.startTime(timestamp=now):

Starts a chronometer for this statistic.

Statistic.sum():

Get the current sum, or zero if values have not been added.

Statistic.toString(sep='\t'):

Prints statistic's id, count, minimum, average, maximum and standard deviation, separated by tabs.

Statistic.variance(center=average):

Calculates current variance, as the average squared difference of each element with the center, which is equal to the average by default.

Returns zero if values have not been added.

new Statistics():

Bundle of Statistic objects by name.

Statistics.__id__(keys):

Generates an id for a Statistic object with the given keys.

new Verifier(logger=Logger.ROOT):

Unit testing helper constructor.

Verifier.assert(condition, message...):

Checks the condition and if it is false raises a failure.

Verifier.assertAlmostEqual(expected, obtained, threshold, msg...):

Asserts that both numbers' difference is not greater than threshold.

Verifier.assertDefined(obj, msg...):

Asserts that the object is not undefined.

Verifier.assertEqual(expected, obtained, msg...):

Asserts that both values are equal (==).

Verifier.assertFails(f, errorType=Error):

Runs the f (as a parameterless function) and fails if it executes without throwing any exceptions. If the execution fails, catches the error and returns it.

Verifier.assertFalse(condition, message...):

Analogue of assert but negating the condition.

Verifier.assertInstanceOf(constructor, object, msg...):

Asserts that the object is instance of the constructor.

Verifier.assertIsArray(expected, obtained, msg...):

Asserts the obtained value is an array with equal elements to expected.

Verifier.assertIsFunction(value, msg...):

Asserts that the value is a function.

Verifier.assertNotEqual(expected, obtained, msg...):

Asserts that both objects are not equal (!=).

Verifier.assertNotNull(obj, msg...):

Asserts that the object is not null.

Verifier.assertNotSame(expected, obtained, msg...):

Asserts that both objects are not the same value (!==).

Verifier.assertNull(obj, msg...):

Asserts that the object is null.

Verifier.assertSame(expected, obtained, msg...):

Asserts that both values are the same value (===).

Verifier.assertSucceeds(f):

Runs the f (as a parameterless function) and fails if it the execution throws any exception. If the execution succeeds, returns the result.

Verifier.assertUndefined(obj, msg...):

Asserts that the object is undefined.

Verifier.EPSILON=2^(-32):

Default threshold used in for numerical difference in some assert methods (e.g. assertAlmostEqual).

Verifier.fail(message...):

Raises an Error with a isFailure property in true.

Verifier.failure(level, message):

Builds the error instance raised by fail().

Verifier.run():

Runs the tests and logs the result.

Verifier.test(testName, testFunction):

Adds the testFunction to this verifier. If testName is not defined, the testFunction's name is used.

Specific kind of failure indicating there is something pending or missing in the implementation of the test.
